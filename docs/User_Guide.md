# Oasis V3 仿真系统玩家指南

## 🌟 核心更新 (M5: Event-Driven Activation)

本次更新引入了 **"事件驱动激活"** 机制，解决了大规模仿真下的市场迟钝问题。

### 1. 激活漏斗机制 (Two-Stage Activation)
系统不再随机抽签，而是通过智能规则筛选高潜力 Agent：
- **平日模式**: 仅允许**高净值** (现金>30w) 或 **高收入** (月薪>2w) 的 Agent 进场。确保每一枚 Token 都花在有购买力的决策上。
- **战时模式 (Market Pulse)**: 当检测到 **"降价潮"**、**"政策利好"** 或 **"恐慌抛售"** 时，门槛自动降低 (现金>10w)，允许更多刚需入场抢购。

### 2. 开发商特价 (Fire Sale)
开发商现已具备**"价格战"**能力。
- 当您在后台大幅降价 (例如 -20%) 时，系统会生成带有 **"🔥超值特价"** 标签的市场公报。
- 这将触发 **Market Pulse**，瞬间唤醒大量潜在买家，引发抢购潮。

## 🎮 快速开始 (Quick Start)

### 启动仿真
```bash
python real_estate_demo_v2_1.py
```
- **Mode 1**: 新开局 (New Game)
- **Mode 2**: 继续存档 (Resume)

### 推荐配置
- **Agent 数量**: 建议 100 - 1000 (系统会自动处理性能优化)。
- **时长**: 建议至少运行 6-12 个月以观察完整周期。

---

## 🛠️ 干预面板 (Researcher Panel)

每月初，您可以通过命令行面板上帝视角干预市场：

1.  **投放房产 (Supply Shock)**: 向 A/B 区投放新房源，可自定义价格。
2.  **调整薪资 (Wage Shock)**: 模拟经济繁荣/萧条。
3.  **调整利率 (Rate Shock)**: 模拟央行加息/降息。
4.  **人口迁移 (Migration)**: 模拟抢人政策。

**💡 玩法建议**: 尝试先"大额投放"导致积压，然后"大幅降价"触发抢购潮，观察 Agent 的群体行为。

## 📊 结果解读

仿真结果保存在 `results/` 目录下：
- `simulation.db`: 完整数据库 (可用 SQLite 浏览器查看)。
- `metadata.json`: 本次运行的配置快照。
- `runner.log`: 详细运行日志 (包含 "Activation Candidates" 筛选数据)。
