# 中国住房市场推演发布操作手册（2026-04-18）

这份手册面向两类人：

1. 第一次上手的外部用户
2. 需要快速复现实验的合作方

目标只有一个：

**让用户不用看代码，也知道从哪里启动、怎么选参数、跑完后去哪里看结果。**

---

## 1. 先理解两个最重要的口径

### 1.1 这里统一叫“回合”

本项目里的一个回合，不等于现实自然月。

一个回合里，会连续发生：

1. 宏观环境更新
2. 家庭生活事件
3. 谁进场、谁观望
4. 挂牌、调价、匹配、谈判、成交
5. 回合归档和 checkpoint

所以更准确的理解是：

- **回合 = 一个虚拟市场周期**

### 1.2 固定供应盘先锁定，需求倍率再调整

现在不再让用户手工拼供给结构。

统一顺序是：

1. 先选固定供应盘
2. 再设需求倍率
3. 再决定是否加入自动冲击

## 3. 固定供应盘怎么选

当前对外开放两类固定供应盘：

1. 梭子型固定供应盘
2. 金字塔型固定供应盘

并且每类都提供：

1. 最小样本
2. 中样本
3. 大样本

### 3.1 梭子型更适合看什么

梭子型更适合看：

1. 主流中段供给更厚时，热点怎么集中
2. 后半段薄市化会不会更快出现
3. 供给承接是否会更快被吃薄

### 3.2 金字塔型更适合看什么

金字塔型更适合看：

1. 入门层和承接层更厚时，后半段会不会更稳
2. 缺口会不会更分散，而不是压成单点瓶颈
3. 供给结构变化会不会改变后半段市场形状

### 3.3 样本量怎么选

最简单的建议：

1. 最小样本
   - 用来排查链路和快速回归
   - 速度最快
   - 但随机波动也最大
2. 中样本
   - 用来做大多数日常验证
   - 速度和稳定性比较平衡
3. 大样本
   - 用来做长测和正式对照
   - 最稳，但也最慢

## 4. 需求倍率怎么设

需求倍率允许范围：

- `0.10x - 2.00x`

可以这样理解：

1. `1.00x`
   - 需求侧人数大致和固定供应套数同量级
2. `0.80x`
   - 需求偏弱，更适合看偏买方市场
3. `1.30x`
   - 需求偏强，更适合看偏卖方市场

系统会自动保障两件事：

1. 每类买家画像都不会被压到消失
2. 每类供应画像都仍有对应买家覆盖

如果你给的倍率太低，系统会自动抬到覆盖下限，并在启动卡里提示。

## 5. 自动冲击怎么用

现在支持三类预排冲击：

1. 收入冲击
2. 增供
3. 减供

典型用法是：

1. 指定在哪一回合触发
2. 指定冲击对象
3. 让系统在该回合开始时自动执行

建议只做小步测试，不要一开始就堆很多动作。

## 6. 跑完以后去哪里看结果

最重要的是三个位置：

### 6.1 主数据库

每轮跑完后，CLI 和日志里都会显示：

- 当前数据库位置

这是最重要的结果入口。

### 6.2 `smoke_report.md`

用途：

1. 快速确认有没有跑完
2. 看 checkpoint 是否完整
3. 看这轮的基础参数

### 6.3 `monthly_checkpoints`

用途：

1. 看每个回合结束时的库快照
2. 复查某一回合到底发生了什么

## 7. 回合末人工干预面板怎么理解

如果系统判断市场明显变薄，会在回合末暂停。

玩家会看到：

1. 热销画像
2. 偏冷画像
3. 缺口画像
4. “有人想买但没房可买”的提示

然后玩家再决定：

1. 定向增供
2. 自动补供
3. 全量补充
4. 减供
5. 强制挂牌

这里最重要的理解是：

- **面板不是装饰，它会真实影响下一回合。**

## 8. 三种推荐玩法

### 8.1 想快速看链路

推荐：

1. 选最小样本
2. 需求倍率 `1.00x`
3. 跑 `3` 回合

### 8.2 想看结构差异

推荐：

1. 分别跑：
   - 梭子型中样本
   - 金字塔型中样本
2. 需求倍率都用 `1.00x`
3. 跑 `6` 回合

### 8.3 想看市场强弱差异

推荐：

1. `buyer_market`
   - `0.80x`
2. `seller_market`
   - `1.30x`
3. 都用 `spindle_large`
4. 跑 `6` 回合

## 9. 最后怎么判断一轮结果值不值得看

优先看这几件事：

1. 有没有完整跑完
2. checkpoint 是否齐全
3. 后半段是不是很早就出现大量“没房可买”
4. 热点是不是只停留在局部假热
5. 玩家在回合末补供后，下一回合有没有真实改善
