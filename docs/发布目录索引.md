# 发布目录索引

这份文档是公开版仓库的目录地图。

它不解释全部内部方法论，只回答三件事：

1. 这个仓库里保留了什么
2. 这个仓库刻意没有保留什么
3. 第一次打开时应该先看哪里

补充一句很重要的话：

**当前公开仓并没有和最新内部工作分支完全同步。**

尤其是：

1. 固定供应盘发布入口
2. 需求倍率发布入口
3. 梭子型 / 金字塔型公开切换
4. 对外全面“回合化”的展示链

这些内容在最新内部发布分支里已经成型，但当前公开仓仍保留一部分旧公开口径和旧入口。

---

## 1. 顶层目录

### `assets/`

这里放公开展示用的图：

- 一页卖点图
- CLI 展示图
- 公开版结果图

### `config/`

这里保留的是公开运行所需的最小配置集合。  
更深的商业场景模板和扩展实验模板，不在公开版仓库中提供。

### `docs/`

这里保留的是公开版说明文档。  
当前重点保留：

- 仓库目录地图
- Scholar CLI 公开复现实验说明
- GitHub 首页 README
- 克隆后最小自检入口说明

### `evidence/`

这里放的是**公开版派生证据**，不是完整原始证据包。  
适合用于快速说明：

- 公开版到底证明了什么
- 公开版保留了哪些可核验结果

### `services/`

服务层代码，包括：

- agent 服务
- transaction 服务
- 状态恢复相关逻辑

### `scripts/`

脚本层，保留公开运行和编排会用到的基础脚本。

当前值得注意的有：

- `public_smoke_test.py`
  - 用于验证新 clone 的仓库能不能启动、跑通 1 回合、写出数据库和报告
- `browser_smoke.js`
  - 用于前端浏览器链的 smoke

### `results/`

公开版仓库中不再保留完整内部正式批次证据。  
完整原始 run 包已经从公开仓库移除。

## 4. 为什么公开版没有保留完整结果批次

因为这个仓库采用的是：

**公开派生证据 + 私有原始证据** 的发布策略。

这样做的目的，是在两个目标之间取平衡：

1. 让外部读者看得懂，也能建立一定信任
2. 保护更完整的内部证据库、治理参数包和商业化沉淀

所以你在这里能看到：

- 公开结果图
- 公开证据摘要
- 可运行入口
- 可复现实验参数说明

但你不会在这里看到：

- 完整 `.db` 批次包
- 更深的内部治理文档
- 商业化场景模板

你同样也不应该假设：

- 这里的对外术语已经和最新内部发布分支完全一致
- 这里的启动入口已经和最新内部发布分支完全一致
